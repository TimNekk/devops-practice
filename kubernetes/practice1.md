## Задание 1: Поды

1. Развернуть Pod с nginx
2. Вывести список всех подов
3. Попробовать kubectl describe
4. Зайти внутрь контейнера и выполнить `curl localhost`
5. Посмотреть логи пода
6. Удалить под

**Дополнительно:**
- Развернуть Pod с Redis
- Подключиться к redis-cli и выполнить команды SET/GET

## Задание 2: Деплойменты

1. Создать Deployment с nginx:1.25, 3 реплики
2. Проверить, что все 3 пода запущены
3. Увеличить количество реплик до 5
4. Удалить один из подов вручную, убедиться что он пересоздался
5. Уменьшить количество реплик обратно до 3

**Дополнительно:**
- Обновить версию образа на nginx:1.26
- Посмотреть историю деплоймента
- Откатить деплоймент на предыдущую версию

## Задание 3: Сервисы

1. Создать Service ClusterIP для деплоймента нджинкса
2. Запустить под с busybox и проверить доступность nginx через Service
3. Создать Service типа NodePort для nginx
4. Открыть nginx в браузере через NodePort

**Дополнительно:**
- Создать два деплоймента: nginx и redis
- Создать Service для redis
- Настроить nginx так, чтобы он мог подключиться к redis по имени сервиса

## Задание 4: ConfigMaps и Secrets

1. Создать ConfigMap с кастомным index.html для nginx
2. Создать Pod с nginx, примонтировать ConfigMap как volume в `/usr/share/nginx/html`
3. Проверить, что nginx отдает кастомную страницу
4. Создать Secret с паролем для базы данных
5. Развернуть Pod с postgres, передать пароль через Secret как переменную окружения

**Дополнительно:**
- Обновить содержимое конфигмапы и проверить, обновился ли файл в поде
